# 项目协作规则

本文件是企业知识平台项目的长期协作规则。后续本地开发、文档编写、提交和同步都按这里执行。

## 文档语言

- 项目文档默认使用中文。
- 技术关键字、代码标识符、命令、API 路径、环境变量名可以保留英文。
- 新增设计文档、实施计划、README、开发说明、部署说明时都使用中文。

## GitHub 同步

- GitHub 仓库固定为 `https://github.com/ming7435/enterprise-knowledge-platform`。
- 本地每次完成项目改动后，都要提交并推送到 GitHub。
- 默认同步目标是 `origin/main`，除非用户明确指定其他分支或暂不推送。
- 提交前必须检查 `git status` 和实际变更范围，不能把无关文件一起提交。

## 隐私与安全

- 不上传账号、密码、API Key、Token、私钥、证书、真实 `.env`、数据库备份、聊天隐私数据。
- 示例配置使用 `.env.example`，真实本地配置使用 `.env`。
- 提交前必须确认没有敏感配置、个人账号和密码进入暂存区。
- `.gitignore` 中列出的敏感文件、缓存、构建产物和本地模型目录不能提交。

## 本地模型

- Rerank 模型固定路径：`L:\RAG_系统\models`。
- 该路径通过环境变量 `RERANK_MODEL_PATH` 配置。
- `models/` 是本地大模型目录，不上传 GitHub。

## 版本推进

- V1：基础可运行版。
- V2：文档与知识库版。
- V3：RAG 问答版。
- V4：企业协作版。
- V5：商业高级版。

每个版本完成后都要提交并同步到 GitHub。
