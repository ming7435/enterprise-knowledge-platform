# 企知云全页面 Dify 式重构设计规格

## 1. 设计目标

本次重构统一“企知云企业知识平台”的全部页面，不新增跨工作区数据流，不改变现有业务接口语义。视觉、布局和交互密度严格采用 Dify 类产品设计语言，解决当前页面之间风格割裂、侧边栏过重、卡片过多、按钮尺寸不一致、信息层级不稳定的问题。

最终目标：

- 登录后形成连续的“工作台 → 知识库 → 智能问答 → 知识图谱”使用路径。
- 个人工作区与企业工作区共用同一套设计系统，但菜单和数据权限继续严格隔离。
- 所有背景、表面、边框、按钮、表格、弹窗、抽屉、状态和响应式规则保持一致。
- 保留企知云名称、Logo、中文文案和业务能力，不使用 Dify 商标、Logo、插图或专有素材。
- 不破坏登录、注册、工作区选择、文档处理、RAG 问答、成员管理、审计日志、模型配置和 Neo4j 图谱功能。

参考资料：

- Dify 官方仓库：<https://github.com/langgenius/dify>
- Dify 官方知识库流程：<https://dify.ai/blog/create-knowledge-in-dify-for-beginners>

## 2. 强制视觉规则

### 2.1 色彩

全站采用 Dify 式低饱和中性色和蓝色主操作色：

| 用途 | 色值 |
| --- | --- |
| 页面背景 | `#F9FAFB` |
| 一级内容面 | `#FFFFFF` |
| 次级背景 | `#F2F4F7` |
| 主文字 | `#101828` |
| 次级文字 | `#344054` |
| 辅助文字 | `#667085` |
| 弱提示文字 | `#98A2B3` |
| 默认边框 | `#E4E7EC` |
| 强边框 | `#D0D5DD` |
| 主操作 | `#155EEF` |
| 主操作悬停 | `#004EEB` |
| 成功 | `#12B76A` |
| 警告 | `#F79009` |
| 错误 | `#D92D20` |

页面主体不得使用现有大面积深色背景、青绿色底色或装饰性渐变。深色只允许用于 Logo、文字、必要的图形可视化和 3D 图谱视口。

### 2.2 字体与尺寸

- 中文字体：`Inter, Noto Sans SC, Microsoft YaHei, sans-serif`。
- 正文：`14px`，辅助信息：`12px`，表格元信息：`11px`。
- 页面标题：`24px` 至 `28px`，区块标题：`14px` 至 `16px`。
- 字距固定为 `0`，不用负字距，不随视口宽度缩放字号。
- 正文行高 `1.5` 至 `1.7`，长回答最大阅读宽度约 `720px`。

### 2.3 间距、圆角和阴影

- 使用 4px 间距基线，常用间距为 `8 / 12 / 16 / 20 / 24 / 32px`。
- 普通控件圆角 `6px`，内容容器 `8px`，大型独立区域不超过 `10px`。
- 按钮高度 `36px` 或 `40px`，输入框高度 `38px` 或 `40px`。
- 图标按钮固定 `36 × 36px`。
- 普通区域优先使用背景、间距和分隔线，不使用卡片套卡片。
- 阴影仅用于下拉菜单、弹窗、抽屉和悬浮层；页面区块默认无阴影。

## 3. 全局信息架构

### 3.1 顶部主导航

所有登录后页面固定使用 64px 顶部导航：

- 左侧：企知云 Logo、当前工作区名称和工作区切换入口。
- 中部：工作台、知识库、智能问答、知识图谱；企业工作区额外显示成员与权限、审计。
- 右侧：全局搜索、帮助、通知和用户头像菜单。
- 当前菜单使用浅灰背景高亮，不使用大面积彩色选中块。

个人工作区不显示成员管理和企业审计入口。企业工作区不读取个人工作区数据。工作区类型判断继续以真实 `workspace.type` 为准。

### 3.2 页面级二级侧栏

二级侧栏仅在需要上下文导航时出现，不作为全站固定主导航：

- 知识库：状态、上传人和隔离范围筛选。
- 智能问答：会话历史。
- 企业设置：设置分组。
- 成员与权限：部门或角色筛选。
- 审计日志：事件类型和时间筛选。

工作台首页、登录页和工作区选择页不显示二级侧栏。

### 3.3 页面头部

页面头部统一包含：

- 11px 英文或中文眉题，可选。
- 24px 页面标题。
- 一句 12px 至 14px 的用途说明。
- 右侧一至两个主要操作。
- 工作区隔离说明放在标题下方，使用绿色圆点和文本，不再制作大型提示卡。

## 4. 页面设计

### 4.1 登录、注册和忘记密码

- 页面背景统一为 `#F9FAFB`，中央表单宽度约 420px。
- Logo、平台名称、表单标题、登录方式切换和表单字段垂直排列。
- 密码登录和验证码登录使用分段控件，不再使用两层标签页。
- 密码输入框保留显示/隐藏按钮；验证码发送按钮位于输入框右侧。
- 忘记密码使用同一表单容器切换，不跳出视觉体系。
- 错误信息显示在对应字段下方；全局错误显示紧凑红色提示条。
- 登录背景不再使用整屏办公室照片，避免入口页与登录后产品风格断裂。

### 4.2 工作区选择

- 顶部保持 Logo、用户和退出入口。
- 主体最大宽度 1120px，个人空间与企业空间以紧凑列表或并列独立对象展示。
- 每个工作区显示名称、类型、角色、文档数量和最近更新时间。
- “进入”是主要操作，“删除”收纳到更多菜单并二次确认。
- 创建企业工作区使用标准弹窗；创建个人工作区仅在不存在时显示。

### 4.3 个人和企业首页

- 结构采用已确认的 Dify 式工作台样稿。
- 首屏只保留欢迎信息、隔离说明、上传文档和创建知识库/开始问答。
- 问答输入区是首页主操作，知识库健康度作为右侧辅助信息。
- 指标使用一条连续统计带，不把每个指标做成独立浮动卡片。
- 最近知识库使用两列紧凑列表；最近动态使用单列时间流。
- 企业首页不再重复显示“最近操作”与“审计摘要”，动态入口统一指向审计日志。
- 个人首页只显示当前个人空间推导出的最近操作，不出现成员或企业审计信息。

### 4.4 文档管理

- 文档管理与知识库共享页面结构，但保留上传进度、重新解析和删除能力。
- 顶部操作为“上传文件”和“批量导入”；支持一次选择多个文件。
- 列表使用 Dify 式数据表格：选择框、文件、上传人、解析状态、入库状态、片段数、更新时间和操作。
- 批量操作只在选择文档后启用；批量删除和单个删除均使用统一确认弹窗。
- 文件名打开右侧详情抽屉，展示摘要、全文、处理状态、失败原因和操作。
- 所有筛选提供“重置”，重置后恢复真实完整列表。

### 4.5 知识库

- 每个文件继续对应一个独立知识库，库名默认为文件名。
- 左侧二级栏提供全部、可检索、处理中、失败、上传人和当前工作区范围。
- 主区域使用统一表格，不再同时展示多套重复卡片。
- 打开知识库后进入详情页，页签为：文档全文、知识片段、检索测试、索引设置。
- 片段默认只展示摘要；完整内容使用详情抽屉。
- 检索设置采用 Dify 式分组：索引质量、向量/全文/混合检索、Top K、相似度阈值和 Rerank。
- 个人与企业知识库查询始终携带当前 `workspace_id`。

### 4.6 智能问答

- 使用三栏结构：左侧会话历史、中间问答、右侧引用来源。
- 顶部显示会话标题和当前真实模型。
- 输入框上方显示已选知识库标签，可多选、移除或关闭知识库进入普通对话。
- RAG 回答使用可读段落、列表和代码块，不展示未经排版的长纯文本。
- 引用来源统一适配 `sources`、`references`、`chunks` 和 `citations`。
- 引用卡显示文件名、片段编号、摘要和真实相似度；无相似度时隐藏该字段。
- 未返回引用时明确显示“本次回答未返回引用来源”，不得伪造。
- 删除会话、删除单条问答和清空历史均二次确认。

### 4.7 知识图谱与高级驾驶舱

- 页面外壳、顶部导航、背景、按钮和面板严格保持 Dify 式设计。
- 默认 2D 图谱使用 `#F9FAFB` 浅色画布、细灰色关系线和类型色节点。
- 3D 模式允许在图形视口内使用深色空间背景，但不能让整页变成另一套深色系统。
- 图谱控制拆分为顶部工具栏、左侧筛选抽屉和右侧实体详情抽屉。
- 两侧面板支持隐藏和拖动调节宽度，图形画布始终自适应剩余空间。
- 节点点击区域大于可见节点，选中后显示描边、邻居高亮和真实来源证据。
- 2D/3D、缩放、居中、刷新、全屏使用固定图标按钮和工具提示。
- 实体和关系来自文件内容，不使用“文档”“对话”“工作区”等界面对象冒充知识实体。
- 高级驾驶舱的数据分析区域使用 Dify 式统计带、列表和轻量图表，不使用深色仪表盘背景。

### 4.8 设置

个人设置二级栏：

- 基础信息
- 模型配置
- 向量检索
- 存储配置
- 安全隔离

企业设置二级栏：

- 基础信息
- 模型 API
- 权限策略
- 存储配置
- 安全隔离

配置页面使用两栏表单或单列分组，不用大量统计卡片。保存按钮固定在内容区底部操作栏。API Key 只显示掩码，真实值不回显、不写日志、不进入前端持久化存储。

### 4.9 成员与权限

- 仅企业工作区显示。
- 顶部提供成员搜索、角色筛选、部门筛选和添加成员。
- 主体使用成员表格，头像、姓名、邮箱、部门、角色、状态、加入时间和操作清晰分列。
- 角色和部门编辑使用下拉菜单；移除成员二次确认。
- 无权限用户显示完整的 permission denied 页面，不白屏、不隐藏错误原因。

### 4.10 审计日志

- 仅企业工作区显示。
- 左侧二级栏提供事件类型和时间范围；顶部提供搜索、刷新、批量删除和保留周期。
- 时间统一为北京时间，格式为 `YYYY-MM-DD HH:mm:ss`。
- 表格列为时间、操作人、事件、对象、状态、来源和操作。
- 单条删除、全部删除和按保留周期删除均使用统一确认弹窗。
- 首页不再保留重复的企业最近操作模块。

## 5. 公共组件边界

重构后按职责拆分以下公共组件：

- `AppHeader`：顶部主导航和工作区入口。
- `ContextSidebar`：页面级筛选和上下文导航。
- `PageHeader`：标题、说明、隔离提示和操作。
- `Button`、`IconButton`、`SegmentedControl`、`Select`、`SearchInput`。
- `StatusBadge`：处理中、成功、失败、禁用和权限状态。
- `DataTable`：桌面表格、选择、批量操作和窄屏横向滚动。
- `EmptyState`、`ErrorState`、`PermissionDenied`、`Skeleton`。
- `Modal`、`Drawer`、`ConfirmDialog`、`Toast`。
- `KnowledgeSelector`、`ChatComposer`、`CitationPanel`。
- `GraphToolbar`、`GraphFilterDrawer`、`GraphInspectorDrawer`。

现有 `WorkspaceShell.tsx`、`PersonalWorkspace.tsx` 和 `EnterpriseWorkspace.tsx` 体积过大。实施时只拆分与页面展示和状态管理直接相关的部分，不修改后端 API 契约，不进行无关业务重写。

## 6. 数据流与隔离

- 登录、注册和验证码接口保持不变。
- 工作区选择后，所有页面从当前 `workspace.id` 读取数据。
- 文档上传、删除、重新解析、检索、问答、图谱、成员和审计操作继续使用真实后端接口。
- 个人和企业组件不共享业务数据，只共享无状态视觉组件。
- 不新增个人与企业之间的同步、复制、导入或共享入口。
- 切换工作区时清理当前页面选择、会话临时状态、筛选和缓存，重新加载目标空间数据。

## 7. 异步状态与错误处理

所有异步操作必须具备：

- 初次加载 Skeleton。
- 局部刷新 Loading。
- 操作成功 Toast。
- 可理解的错误提示和重试按钮。
- 空数据 EmptyState。
- 条件不满足时 Disabled 状态和原因提示。
- 权限不足 PermissionDenied。

禁止接口失败后保留空白容器、静默失败或让整个页面崩溃。

## 8. 响应式规则

- `>= 1280px`：完整顶部导航和多栏布局。
- `1024px - 1279px`：隐藏非关键导航文字，表格允许横向滚动。
- `768px - 1023px`：二级侧栏变为抽屉，首页双列改为单列。
- `< 768px`：顶部导航收纳到菜单；智能问答引用栏改为底部抽屉；所有操作按钮保证可点击且文字不裁切。
- 知识图谱画布使用稳定的最小高度和自适应宽度，面板收起后必须重新计算视口。

## 9. 测试与验收

### 9.1 自动检查

- 前端 TypeScript 编译通过。
- 前端生产构建通过。
- 后端 pytest 全量通过。
- 工作区隔离相关测试继续通过。

### 9.2 页面检查

逐页检查：

- 登录、注册、验证码登录、忘记密码。
- 工作区选择、创建和删除。
- 个人区 6 个页面。
- 企业区 8 个页面。
- 文档上传、批量上传、重新解析、批量删除和详情。
- 知识库筛选、重置、全文、片段和检索。
- 普通问答、RAG 问答、知识库多选、模型显示和引用来源。
- 图谱 2D、3D、节点选择、详情、筛选、面板隐藏和宽度调整。
- 成员添加、角色修改和删除。
- 审计筛选、单条删除、全部删除和保留周期删除。
- 模型配置保存、连接测试和错误提示。

### 9.3 视觉检查

使用桌面宽屏、普通笔记本和窄屏三种视口截图，对比以下内容：

- 页面背景、顶部导航、二级侧栏和内容面一致。
- 按钮、输入框、状态标签、表格、弹窗和抽屉尺寸一致。
- 无文字裁切、重叠、横向溢出和错误的卡片嵌套。
- 2D/3D 图谱非空白、节点可点击、画布不被面板遮挡。

## 10. 实施顺序

本次全页面重构按一个设计系统、四个实施阶段推进：

1. 建立设计令牌、基础组件、顶部导航和页面外壳。
2. 重构入口页、工作区选择、个人工作区页面。
3. 重构企业工作区页面、成员、审计和设置。
4. 重构知识图谱与高级驾驶舱，完成响应式和全量回归。

每一阶段都必须保持项目可构建、现有业务接口可用，并在本地验证后同步服务器验证。
