# 企业知识平台项目功能与实现清单

更新时间：2026-07-07
适用目录：本地 `L:\RAG_系统`，服务器 `/opt/enterprise-knowledge-platform/RAG_系统`

## 1. 项目定位

本项目是一个支持个人工作区与企业工作区隔离使用的企业知识平台。

核心入口顺序固定为：

1. 注册或登录。
2. 登录后进入工作区选择页。
3. 选择个人工作区或企业工作区。
4. 进入对应工作区后，只显示该工作区类型可用的菜单和功能。

核心隔离原则：

- 个人工作区数据只属于当前用户个人空间。
- 企业工作区数据只属于当前企业空间和有权限的成员。
- 个人工作区和企业工作区之间不做同步、复制、导入、共享。
- 所有文档、知识片段、问答、设置、审计日志接口都必须携带或使用当前 `workspace_id`。

## 4. 登录、注册、验证码和密码重置

### 4.1 页面入口

实现文件：

- `frontend/src/App.tsx`
- `frontend/src/components/AuthForms.tsx`
- `backend/app/api/v1/auth.py`
- `backend/app/services/auth_service.py`

页面状态：

- 未登录时只显示登录/注册页面。
- 登录成功后进入工作区选择页。
- 已选择工作区后进入对应工作区页面。

### 4.2 密码登录

位置：

- 登录页。
- 顶部模式选择为“登录”。
- 登录方式选择为“密码”。

字段：

- 邮箱。
- 密码。
- 密码输入框右侧有“可见/隐藏”按钮。

按钮：

- `登录`：提交邮箱和密码。
- `忘记密码?`：进入密码重置流程。

接口：

- `POST /api/v1/auth/login`

实现：

- 前端调用 `api.login()`。
- 后端校验邮箱和密码。
- 成功后返回 JWT 和用户信息。
- 前端保存 `ekp_token`，进入工作区加载流程。

失败状态：

- 密码错误显示错误信息。
- 后端不可达时显示请求失败。

### 4.3 邮箱验证码登录

位置：

- 登录页。
- 顶部模式选择为“登录”。
- 登录方式选择为“验证码”。

字段：

- 邮箱。
- 邮箱验证码。

按钮：

- `发送`：发送登录验证码。
- `验证码登录`：用邮箱验证码登录。

接口：

- `POST /api/v1/auth/email-code/send`
- `POST /api/v1/auth/email-code/login`

实现：

- 验证码用途为 `login`。
- 后端检查邮箱是否存在。
- 验证码有效期由 `EMAIL_CODE_EXPIRE_MINUTES` 控制。
- 重发间隔由 `EMAIL_CODE_RESEND_SECONDS` 控制。

### 4.4 注册

位置：

- 登录页右侧或顶部模式切换为“注册”。

字段：

- 用户名。
- 邮箱。
- 密码。
- 邮箱验证码。

按钮：

- `发送`：发送注册验证码。
- `注册`：提交注册。

接口：

- `POST /api/v1/auth/email-code/send`
- `POST /api/v1/auth/register`

实现：

- 注册必须通过邮箱验证码。
- 注册成功后创建个人工作区。
- 注册成功后自动登录或进入登录后流程。

### 4.5 忘记密码

位置：

- 密码登录方式下，密码框下方。

字段：

- 邮箱。
- 新密码。
- 邮箱验证码。

按钮：

- `发送`：发送密码重置验证码。
- `重置密码`：提交新密码。

接口：

- `POST /api/v1/auth/email-code/send`
- `POST /api/v1/auth/password/reset`

实现：

- 验证码用途为 `reset`。
- 后端检查邮箱是否存在。
- 重置成功后可返回登录。

位置：

按钮：

- `新建`

- `POST /api/v1/workspaces/personal`

- 为当前用户创建个人工作区。
- 工作区类型为 `personal`。

### 5.4 新建企业工作区

位置：

- 工作区选择页右侧表单。

字段：

- 企业名称。
- 说明。

按钮：

- `创建企业工作区`

接口：

- `POST /api/v1/workspaces/enterprise`

实现：

- 创建企业工作区。
- 当前用户成为企业 owner。

### 5.5 删除工作区按钮

位置：

- 个人工作区卡片右上或右侧垃圾桶图标。
- 企业工作区卡片右上或右侧垃圾桶图标。

按钮：

- 图标按钮，标题为 `删除工作区`。

二次确认：

- `确认删除“工作区名称”吗？工作区内文档、知识片段和会话记录会同步删除。`

接口：

- `DELETE /api/v1/workspaces/{workspace_id}`

实现：

- 仅删除当前选择的那个工作区。
- 后端同时清理该工作区内文档、片段、问答会话、成员或相关记录。
- 不影响其他工作区。

## 10. 数据隔离实现

### 10.1 前端隔离

- 当前工作区保存在前端状态中。
- 所有 API 调用都带 `workspace_id`。
- 菜单按 `workspace.type` 分支展示。
- 个人区不显示成员管理和企业审计日志。
- 企业区不显示个人专属模型和向量库配置。

### 10.2 后端隔离

- 接口通过当前用户和 `workspace_id` 校验成员关系。
- 没有工作区权限时拒绝访问。
- 删除、上传、检索、问答都限定当前工作区。
- 企业成员管理和审计日志只允许企业工作区使用。

### 10.3 数据库隔离字段

主要隔离字段：

- `workspace_id`
- `workspace.type`
- `user_id`
- `document_id`

规则：

- 查询文档必须按 `workspace_id`。
- 查询片段必须按 `workspace_id`。
- 问答检索必须按 `workspace_id`。
- 选择知识库时再追加 `document_ids` 过滤。
- 审计日志必须按企业 `workspace_id`。

## 11. 问答和模型调用实现

### 11.1 普通对话

触发条件：

- `use_knowledge_base=false`

行为：

- 不检索知识库。
- 直接使用当前工作区模型配置调用大模型。
- 返回普通文本回答。
- 问答结果展示当前使用模型。

### 11.2 RAG 问答

触发条件：

- `use_knowledge_base=true`

行为：

- 根据当前 `workspace_id` 检索知识片段。
- 如果选择了文件知识库，只在 `document_ids` 内检索。
- 组装上下文。
- 调用大模型生成正常文本回答。
- 返回引用来源。

### 11.3 引用来源规则

展示字段：

- 来源文档。
- 片段编号。
- 相似度。
- 内容摘要。

兼容字段：

- `sources`
- `references`
- `chunks`
- `citations`

没有引用时：

- 显示“本次回答未返回引用来源”。

### 11.4 模型 API 配置

个人区：

- 默认 DeepSeek。
- 可配置模型供应商和模型版本。
- 可配置 RAG、引用、温度、最大长度。
- 可配置向量检索参数。

企业区：

- 提供模型 API 配置。
- 企业可配置 API Key、Base URL、供应商、模型版本。
- API Key 脱敏展示。

注意：

- 没有真实 API Key 时，测试连接和真实模型调用应失败并提示原因，不能伪造成功。

## 13. 文件知识库实现

产品规则：

- 每个上传文件就是一个知识库。
- 知识库名称等于文件名。
- 知识库内包含该文件生成的所有知识片段。
- 可以查看整个文档。
- 可以查看该文档全部片段。
- 可以选择多个文件知识库进入问答。

实现方式：

- 复用已有 `document` 作为文件知识库实体。
- 片段通过 `document_id` 关联。
- 搜索筛选从“全部文档”改为“库名筛选”。
- 上传人筛选使用上传人名称。

删除规则：

- 删除文件知识库等同于删除文档。
- 同时删除该文档片段和相关索引。
